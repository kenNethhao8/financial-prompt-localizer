# Derivatives, Fixed Income, and Credit Risk Prompt Template

```text
Task:
Build a [pricing / risk / simulation] workflow for [derivative / bond / credit portfolio].

Context:
The user wants a financial engineering project with clear model assumptions and validation.

Requirements:
- State model assumptions, inputs, units, and conventions.
- Separate pricing, risk sensitivities, calibration, simulation, and validation.
- Include scenario analysis or sensitivity analysis.
- Avoid implying model outputs are trading recommendations.

Inputs and assumptions:
- Derivatives: spot, strike, maturity, volatility, rate, dividend yield if relevant.
- Fixed income: cash flows, yield, maturity, coupon, day-count, compounding.
- Credit risk: exposure, rating states, transition matrix, recovery rate, default state.

Output format:
- Save pricing tables, risk measures, simulation summaries, and scenario results.

Validation:
- Check formula assumptions, row sums for transition matrices, convergence, put-call parity, duration/convexity approximation, or limiting cases as relevant.
```
